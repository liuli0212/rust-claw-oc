# 🚀 JaviRust `/autopilot` (自动驾驶模式) 实施方案

## 1. 架构总览
本方案旨在实现一个基于“客观物理校验”与“状态外置”的无限续航自动驾驶模式。核心目标是根治大模型“不爱写计划”和“死循环幻觉”的痛点，使其能够独立、稳定地完成长周期复杂工程任务。

## 2. 实施阶段划分与详细设计

### [x] Phase 1: 状态外置与强制规划 (Forced Planning via `TODOS.md`)
**目标**：保留现有 `TaskPlanTool` 供普通模式使用，但在 `/autopilot` 模式下，强制 Agent 使用物理文件 `TODOS.md` 进行任务管理。

*   **1.1 兼容性保留**：
    *   现有的 `TaskPlanTool` 保持不变，普通对��模式依然可以使用。
*   **1.2 动态提示词重构 (`src/context.rs`)**：
    *   当 `is_autopilot == true` 时，在 System Prompt 中注入强指令：“你必须在项目根目录维护一个 `TODOS.md` 文件。使用 `- [ ]` 表示未完成，`- [x]` 表示已完成。在执行任何代码修改前，必须先创建或更新该文件。”
*   **1.3 模式专属拦截器 (The Gatekeeper) (`src/core/step_helpers.rs`)**：
    *   在 `execute_tool_round` 方法中注入逻辑：**仅当处于 autopilot 模式时**，如果调用的工具是**修改类工具**（如 `write_file`, `patch_file`）或**高危命令**（如 `cargo build`, `npm install`），系统检查根目录下是否存在 `TODOS.md`。
    *   **探索宽限期**：允许 Agent 使用只读工具（如 `read_file`, `ls`, `grep`）进行项目探索，不强制拦截。
    *   **拦截**：如果试图修改代码但 `TODOS.md` 不存在，阻断工具执行，并向 LLM 返回错误：`[System Error] Action Denied. Autopilot 模式下必须先创建并规划 TODOS.md。`

### [x] Phase 2: 防卡死引擎与反思机制 (Action Hash Wall & Reflection)
**目标**：在常规执行中，实现死循环的“警告-反思-熔断”三级机制。

*   **2.1 状态结构扩展 (`src/core.rs`)**：
    *   在 `AgentLoop` 结构体中新增字段：`action_history: VecDeque<u64>` 和 `reflection_strike: u8`。
*   **2.2 哈希计算与反思触发 (`src/core/step_helpers.rs`)**：
    *   **降级哈希维度**：为了避免时间戳陷阱，每次工具执行完毕后，计算 `Hash(ToolName + Args + ExitCode)` 并压入队列。或者在 Hash 前使用正则剔除 `ErrorOutput` 中的时间戳和 ANSI 颜色码。
    *   **一级警告（反思）**：如果发现连续 3 次出现完全相同的 Hash，不直接退出，而是向 LLM 注入一条系统级 SystemMessage：“[System Warning] 检测到你正在重复执行相同的错误动作。请立即停止当前尝试，反思失败原因，并提出全新的解决路径。”
    *   **二级熔断（交还用户）**：如果反思后继续执行，再次触发连续 3 次相同 Hash（即 `reflection_strike >= 2`），则抛出 `RunExit::RecoverableFailed("检测到深度死循环，反思无效，交还控制权")`，强制暂停。

### [x] Phase 3: 触顶重置与滚动摘要 (Checkpoint & Rolling Summary)
**目标**：当 Energy（默认 25 轮）耗尽时，通过物理审计决定是否续航，并用结构化的 Summary 替代完全失忆。

*   **3.1 开启自动驾驶模式 (`src/app/cli.rs` & `src/core.rs`)**：
    *   CLI 新增 `/autopilot <goal>` 命令。
    *   `AgentLoop` 新增标志位 `is_autopilot: bool`。
*   **3.2 物理足迹审计 (Physical Audit)**：
    *   当 `energy_points == 0` 且 `is_autopilot == true` 时，触发审计。
    *   **收紧审计标准**：物理审计**只认 `TODOS.md`**。必须通过正则严格校验 `TODOS.md` 中的 `[x]` 数量是否比 25 轮前增加了。没打勾，就不给续航。
*   **3.3 结构化滚动摘要续航 (Structured Rolling Summary) (`src/context.rs`)**：
    *   如果审计通过：
        1.  **不完全清空上下文**。调用一个**极速、便宜的副脑模型**（如 Gemini Flash 或 OpenRouter Free）对过去 25 轮的执行轨迹生成一段精简的 Summary（包含已尝试的路径和关键错误），以节省成本并防止主模型“粉饰太平”。
        2.  **【设计优化】**：不使用伪造的 `SystemMessage`。而是在 `AgentContext` 中新增一个 `rolling_summary` 字段。在 `ContextAssembler` 组装 Prompt 时，将其作为一个独立的上下文块（如 `--- [PREVIOUS AUTOPILOT SUMMARY] ---`）注入，保持用户与 LLM 交互角色的纯净性。
        3.  将 `energy_points` 重新充满。
        4.  静默继续执行 `loop`。
    *   如果审计失败：返回 `RunExit::StoppedByUser("自动驾驶终止：过去 25 轮未检测到任何物理进展。")`。

### [x] Phase 4: 清晰的进度播报与安全退出 (Clear Notifications & Exit)
**目标**：完善退出条件，并向用户提供清晰、易懂的进度报告。

*   **4.1 胜利退出条件**：
    *   Agent 主动调用 `finish_task`。
    *   系统在 `finish_task` 拦截器中额外校验：`TODOS.md` 中是否还有未完成的 `- [ ]`。如果有，拒绝 finish 并要求其继续执行。
*   **4.2 进度播报 (`src/core.rs`)**：
    *   **【设计优化】**：不追求极简，而是追求“清晰”。当系统检测到 `TODOS.md` 被修改时，系统在底层解析 Markdown，提取出刚刚被勾选的项和下一个待办项。
    *   通过 `self.output.on_text()` 向用户打印结构化报告，例如：
        `[Autopilot 进展] ✅ 已完成: "修复 auth.rs 编译错误" | ⏳ 下一步: "运行 cargo test"`

### [ ] Phase 5: 人在回路与中断恢复 (Human-in-the-loop & Resume)
**目标**：允许用户在自动驾驶过程中随时打断、纠偏，并无缝恢复执行或降级为手动模式。

*   **5.1 优雅熔断与友好提示 (Graceful Abort & Friendly Prompt)**：
    *   用户通过 `Ctrl-C` (CLI) 或“停止任务”按钮 (Telegram) 触发中断。
    *   **【安全权衡】**：对于正在执行的 PTY 子进程（如 `cargo build`），系统不使用暴力的 `SIGKILL`，而是发送标准的 `SIGINT` (Ctrl+C 信号)，允许子进程优雅退出，防止文件损坏或锁残留（如 `.git/index.lock`）。
    *   **优雅降级 (Graceful Degradation)**：发送 `SIGINT` 后，系统等待 3 秒。如果 PTY 里的子进程依然存活，则毫不留情地发送 `SIGKILL` 强杀。先礼后兵，确保系统绝对控制权。对于 LLM 网络流则直接取消。
    *   `AgentLoop` 退出当前循环，保持 `is_autopilot = true` 状态（挂起）。
    *   **【交互优化】**：向用户打印友好的引导提示：`[Autopilot 挂起] 任务已安全暂停。您可以直接输入指导意见来纠偏并自动继续，或者输入 /manual 彻底退出自动驾驶。`
*   **5.2 上下文注入与纠偏 (Context Injection)**：
    *   用户输入补充信息（如“思路不对，去改 config.rs”）。
    *   该信息作为标准的 `UserMessage` 追加到当前上下文中，让 Agent 明确知道自己被打断及最新的纠偏指导。
*   **5.3 无缝恢复 (Seamless Resume)**：
    *   系统调用 `agent.step()` 处理用户的新输入。
    *   检测到 `is_autopilot == true`，系统自动重新进入自动驾驶的 `while` 循环。
    *   重置 `energy_points`，Agent 带着最新指示继续执行 `TODOS.md` 任务。
*   **5.4 降级逻辑 (Manual Override)**：
    *   新增 `/manual` 指令。
    *   用户输入 `/manual` 时，系统将 `is_autopilot` 设为 `false`，退出自动驾驶模式，退回标准的一问一答模式。
